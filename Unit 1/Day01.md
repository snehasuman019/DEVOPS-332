#  Day 01 — Introduction to DevOps

##  What is DevOps?

DevOps is a **software development approach and culture** that focuses on collaboration between Development (Dev) and Operations (Ops) teams to build, test, deploy, and maintain applications **faster and more reliably.**

Earlier, developers only wrote code and operations teams only handled deployment.  
This created delays, failures, and communication gaps.

DevOps solves this by **bringing both teams together with shared responsibility and automation.**

>  DevOps = Faster Development + Continuous Testing + Reliable Deployment


##  Why DevOps is Needed?

Before DevOps:

- Development and Operations worked separately
- Software releases were slow
- Manual deployment caused errors
- Bug fixing took longer time
- Customer satisfaction was low

After DevOps:

- Teams collaborate closely
- Automation reduces manual work
- Continuous testing improves quality
- Faster and safer deployments
- Quick feedback and improvement


##  Goals of DevOps

- Faster software delivery
- Improved collaboration
- Continuous Integration and Deployment
- Reduced deployment failures
- Faster bug fixing and recovery
- Better product quality
- High system reliability and scalability


## DevOps Lifecycle

DevOps follows a continuous cycle:

1. **Planning** → Requirement gathering and task planning  
2. **Development** → Writing and managing code using version control tools  
3. **Build** → Compiling and packaging the application  
4. **Testing** → Automated testing to ensure quality  
5. **Release** → Preparing the application for deployment  
6. **Deployment** → Releasing application to production servers  
7. **Operations** → Monitoring system performance and stability  
8. **Monitoring & Feedback** → Collecting feedback for improvements  


##  Key Principles of DevOps

###  Collaboration
Development and Operations teams work together throughout the lifecycle.

###  Automation
Manual tasks like testing, building, and deployment are automated.

###  Continuous Integration (CI)
Developers frequently integrate code changes into a shared repository.

###  Continuous Deployment (CD)
Applications are automatically deployed after successful testing.

### Monitoring
System performance and logs are continuously monitored.

###  Feedback Loop
User feedback helps improve the product continuously.


## Benefits of DevOps

- Faster release cycles
- Increased productivity
- Reduced failure rate
- Better teamwork
- Improved system stability
- Faster recovery from issues
- Enhanced customer satisfaction

---

##  Real Life Example

Think of DevOps like a restaurant kitchen.

- Chef (Developer) prepares food  
- Server (Operations) delivers food  

If they don’t coordinate → wrong orders and delays.  
DevOps makes them work as **one team** using automation tools to deliver faster and correctly.
